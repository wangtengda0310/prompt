# 渐进式披露文档维护技能

使用渐进式披露原则维护项目的 CLAUDE.md 层级文档体系，确保 AI 在有限的上下文窗口内高效获取项目知识。

## 核心原则

1. **层级递进**：根文档提供概览和索引，子文档提供细节，叶子文档提供实现
2. **引用而非复制**：文档中引用代码位置（文件:行号），不复制代码片段
3. **可追溯性**：所有文档从根 CLAUDE.md 出发可逐步追溯到达
4. **Token 经济**：每级文档控制大小，避免单文档过长消耗对话 token

## 文档层级定义

### Level 0 — 项目根 CLAUDE.md

**定位**：项目入口，AI 每次对话首先加载

**内容**：
- 项目一句话定位
- 技术栈概览
- 子模块/子目录索引表（名称 → CLAUDE.md 相对路径）
- 关键约束和注意事项
- 开发常用命令

**大小限制**：≤ 200 行

**模板**：
```markdown
# {项目名}

{一句话描述}

## 技术栈
- 后端: {语言} {框架}
- 前端: {框架} {UI库}

## 子模块

| 模块 | 说明 | 文档 |
|------|------|------|
| module-a | 模块描述 | [CLAUDE.md](./module-a/CLAUDE.md) |

## 关键约束
- {重要注意事项}

## 更多文档
- [{文档名}]({相对路径}) — 一句话说明
```

### Level 1 — 模块 CLAUDE.md

**定位**：模块入口，AI 需要深入了解该模块时加载

**内容**：
- 模块定位和职责
- 目录结构树
- 核心概念解释
- 关键入口（文件:行号）
- 子目录/子模块索引表
- 与其他模块的依赖关系

**大小限制**：≤ 150 行

**模板**：
```markdown
# {模块名}

{模块职责描述}

## 目录结构
```
module/
├── subdir-a/     # 说明
├── file-a.go     # 说明
└── ...
```

## 核心概念
- {概念}: {一句话解释}

## 关键入口
| 功能 | 文件位置 |
|------|----------|
| {功能} | {文件}:{行号} |

## 子目录文档
| 目录 | 说明 | 文档 |
|------|------|------|
| subdir-a | 说明 | [CLAUDE.md](./subdir-a/CLAUDE.md) |

## 依赖关系
- 依赖: {其他模块}
- 被依赖: {哪些模块使用本模块}
```

### Level 2 — 子模块/包 CLAUDE.md

**定位**：具体实现细节，AI 需要修改该部分代码时加载

**内容**：
- 数据结构定义（类型名 + 字段说明，不复制完整定义）
- 核心函数/方法列表（名称 + 职责）
- 配置项说明
- 开发注意事项

**大小限制**：≤ 100 行

**模板**：
```markdown
# {子模块名}

{一句话说明}

## 核心类型
| 类型 | 文件位置 | 说明 |
|------|----------|------|
| TypeName | file.go:L{行号} | {说明} |

## 核心函数
| 函数 | 文件位置 | 职责 |
|------|----------|------|
| FuncName | file.go:L{行号} | {说明} |

## 开发注意事项
- {注意事项}
```

### Level 3 — 深度文档 (docs/ 目录)

**定位**：特定功能的详细说明，按需加载

**内容**：
- 实现详解
- 设计决策和原因
- 扩展指南
- 排错手册

**大小限制**：无硬限制，但应聚焦单一主题

**命名规范**：`{功能名}-{文档类型}.md`
- `XXX-实现详解.md`
- `XXX-扩展指南.md`
- `XXX-排错手册.md`

## 文档维护规则

### 何时创建文档

| 触发条件 | 创建级别 | 位置 |
|----------|----------|------|
| 新建项目/模块 | Level 0 | 项目根目录 |
| 新建子目录且有独立职责 | Level 1/2 | 子目录内 |
| 实现复杂功能 | Level 3 | docs/ 目录 |
| 添加新的公共 API | Level 2 | 对应包目录 |

### 何时更新文档

| 触发条件 | 更新内容 |
|----------|----------|
| 新增/删除文件 | 目录结构树 |
| 修改公共 API | 类型/函数列表 |
| 修改架构或流程 | 核心概念、数据流 |
| 新增依赖 | 依赖关系 |
| 发现重要约束 | 注意事项 |
| 修复重要 bug | 开发注意事项 |

### 引用规范

**代码引用**（优先使用）：
```markdown
入口函数 main.go:34
配置定义 internal/config.go:15-30
核心逻辑 见 process() 方法 (service.go:82)
```

**文档交叉引用**：
```markdown
详细说明见 [文档名](相对路径)
更多配置参考 [配置文档](./docs/config.md)
```

**禁止的引用方式**：
```markdown
# ❌ 复制代码片段
func main() {
    app := wails.New()
    // ...
}

# ❌ 模糊引用
见代码中的注释
在某个文件里
```

## 文档索引维护

### 根 CLAUDE.md 的"更多文档"章节

所有 Level 3 文档必须在根 CLAUDE.md 的"更多文档"章节中有索引条目：

```markdown
## 更多文档
- [{标题}]({相对路径}) — {一句话说明}
```

### 索引条目格式规范

```markdown
- [武将Wiki检查-实现详解](docs/武将Wiki检查-实现详解.md) — 快照机制、差异检测、通知流程
```

- 标题用方括号
- 路径用相对路径
- 破折号后是一句话摘要（不超过 30 字）
- 按重要性排序，最常用的在前

## 质量检查清单

创建或更新文档后，逐项检查：

### 结构检查
- [ ] 根 CLAUDE.md ≤ 200 行
- [ ] 模块 CLAUDE.md ≤ 150 行
- [ ] 子模块 CLAUDE.md ≤ 100 行
- [ ] 所有文档可从根 CLAUDE.md 追溯到达

### 内容检查
- [ ] 没有复制代码片段（用文件:行号引用替代）
- [ ] 没有过时的信息（与代码一致）
- [ ] 没有重复内容（同一信息只在一处维护）
- [ ] 交叉引用路径正确

### 渐进式检查
- [ ] 上层文档只包含概览和索引
- [ ] 细节只在叶子文档中出现
- [ ] 每级文档阅读时间 < 2 分钟
- [ ] AI 读完根文档后能判断是否需要加载子文档

## 执行步骤

### 新建文档流程

1. **确定级别**：判断文档属于哪个层级
2. **选择模板**：使用对应层级的模板
3. **填充内容**：基于实际代码填写，引用而非复制
4. **更新索引**：在上级文档中添加引用条目
5. **验证追溯**：从根 CLAUDE.md 出发能找到新文档

### 更新文档流程

1. **定位文档**：找到需要更新的文档
2. **判断范围**：是否需要同步更新上级/下级文档
3. **更新内容**：修改对应章节
4. **验证引用**：确保引用的文件:行号仍然正确
5. **检查重复**：确保没有在多处维护相同信息

### 重构文档流程

当单文档超过大小限制时：

1. **识别可下推的内容**：细节性内容移到子文档
2. **创建子文档**：在对应子目录创建 CLAUDE.md
3. **精简父文档**：只保留概览和引用
4. **更新索引链**：确保追溯路径完整
5. **删除冗余**：移除父文档中的重复内容

## 常见错误模式

| 错误 | 正确做法 |
|------|----------|
| 根文档包含实现细节 | 只保留概览，细节下推到子文档 |
| 复制代码片段到文档 | 引用 `文件:行号` |
| 多个文档维护同一信息 | 信息只在最合适的文档中出现一次 |
| 文档中硬编码具体值 | 引用常量定义位置 |
| 新建文档但不更新索引 | 每次新建后更新上级文档的索引 |
| 文档内容过时未更新 | 代码变更后立即同步文档 |
