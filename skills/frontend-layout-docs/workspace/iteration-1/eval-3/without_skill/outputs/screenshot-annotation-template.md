# 前端布局截图标注说明模板

## 概述

本文档提供前端布局文档中截图标注的标准化模板和最佳实践，确保文档的一致性和可读性。

---

## 一、截图标注基本元素

### 1.1 标注类型

| 标注类型 | 用途 | 颜色建议 | 样式 |
|---------|------|---------|------|
| **区域框** | 标识组件/模块范围 | 红色 `#FF0000` | 实线边框 |
| **编号标记** | 多区域顺序说明 | 蓝色 `#0066CC` | 圆形+数字 |
| **箭头指示** | 指向特定元素 | 橙色 `#FF9900` | 带箭头线条 |
| **文字说明** | 补充描述信息 | 黑色 `#333333` | 白底文字 |
| **连接线** | 关联多个元素 | 绿色 `#33CC33` | 虚线 |

### 1.2 标注规范

```
标注编号格式: [类型缩写]-[序号]

示例:
- A-1, A-2 (Area - 区域)
- C-1, C-2 (Component - 组件)
- F-1, F-2 (Feature - 功能点)
```

## 三、标注最佳实践

### 3.1 截图准备

1. **分辨率要求**
   - 推荐: 1920×1080 或更高
   - 缩放比例: 100% (避免模糊)
   - 格式: PNG (保持清晰度)

2. **截图范围**
   - 完整页面: 包含浏览器边框
   - 组件截图: 预留 20px 边距
   - 局部放大: 标注缩放比例

3. **状态捕获**
   - 默认状态 (必须)
   - 悬停状态 (可选)
   - 激活状态 (可选)
   - 错误状态 (可选)

### 3.2 标注原则

1. **清晰性**
   - 标注不遮挡关键内容
   - 文字大小适中 (12-14px)
   - 颜色对比度足够

2. **一致性**
   - 同一文档使用相同的颜色方案
   - 编号顺序: 从左到右, 从上到下
   - 相同类型元素使用相同样式

3. **简洁性**
   - 每张截图标注不超过 10 个
   - 避免重复说明
   - 复杂内容拆分多张图

### 3.3 工具推荐

| 工具 | 适用场景 | 特点 |
|------|---------|------|
| Snipaste | 快速标注 | 免费易用, 支持贴图 |
| Draw.io | 复杂图表 | 功能全面, 支持导出 |
| Figma | 设计稿标注 | 专业级, 支持协作 |
| Excalidraw | 手绘风格 | 轻量级, 适合草图 |
