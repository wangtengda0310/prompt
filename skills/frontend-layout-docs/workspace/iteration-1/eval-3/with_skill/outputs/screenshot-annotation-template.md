# 截图标注文档模板

> 本文档为前端布局文档提供截图标注的标准模板和最佳实践。

## 目录结构规范

```
docs/
├── layout/
│   ├── CLAUDE.md              # 索引文件
│   └── PageName-layout.md     # 布局文档
└── screenshots/               # 截图目录
    ├── CLAUDE.md              # 截图索引
    └── PageName/              # 按页面名称组织
        ├── overview.png       # 整体截图
        ├── detail-header.png  # 可选：细节截图
        ├── detail-footer.png
        └── annotations.md     # 标注说明文档
```

## 截图命名规范

| 类型 | 命名格式 | 示例 | 说明 |
|------|---------|------|------|
| 整体截图 | `overview.png` | `overview.png` | 页面完整视图 |
| 区域截图 | `detail-{区域名}.png` | `detail-header.png` | 特定区域的详细截图 |
| 状态截图 | `state-{状态名}.png` | `state-loading.png` | 特定状态的截图 |
| 交互截图 | `action-{动作名}.png` | `action-dropdown.png` | 交互效果的截图 |

## 最佳实践

### 1. 截图采集

**工具推荐:**
- Playwright: 自动化截图，适合 CI/CD 集成
- Chrome DevTools: 手动截图，支持设备模拟
- Snipaste: 截图标注一体化工具

**截图要求:**
```javascript
// Playwright 截图示例
await page.goto('http://localhost:3000/page');
await page.waitForLoadState('networkidle');

// 整体截图
await page.screenshot({
  path: 'overview.png',
  fullPage: true  // 完整页面截图
});

// 元素截图
const header = await page.$('.header');
await header.screenshot({ path: 'detail-header.png' });
```

**分辨率标准:**
- 桌面端: 1920x1080 (默认), 1366x768 (兼容)
- 平板端: 768x1024
- 移动端: 375x667

### 2. 标注原则

**清晰性:**
- 使用一致的编号系统 (A, B, C 或 1, 2, 3)
- 标注线不遮挡关键内容
- 颜色对比度足够高

**完整性:**
- 覆盖所有可见的交互元素
- 包含边界情况和异常状态
- 标注尺寸信息

**可维护性:**
- 与代码行号关联
- 使用相对路径引用图片
- 记录截图版本和分辨率

### 3. 区域划分策略

**按功能区域划分:**
```
┌─────────────────────────────────────────────────────────────┐
│ A. Header (顶部导航)                                        │
├────────────────┬────────────────────────────────────────────┤
│ B. Sider       │ C. Content (主内容区)                      │
│ (侧边栏)        │                                            │
│                │ ┌────────────────────────────────────────┐ │
│                │ │ C1. Tab栏                              │ │
│                │ ├────────────────────────────────────────┤ │
│                │ │ C2. 内容面板                           │ │
│                │ └────────────────────────────────────────┘ │
├────────────────┴────────────────────────────────────────────┤
│ D. Footer (底部状态栏)                                       │
└─────────────────────────────────────────────────────────────┘
```

**层级编号规则:**
- 一级区域: 大写字母 (A, B, C, D)
- 二级区域: 字母+数字 (A1, B1, C1)
- 三级区域: 字母+数字+小写 (A1a, B1b)

### 4. 与布局文档的关联

```markdown

```

### 5. 更新维护

**何时更新截图:**
- 页面布局结构发生变化
- 组件位置或尺寸调整
- 新增/删除功能区域
- UI 样式重大变更

**版本管理:**
```markdown
> 截图版本: 2024-01-15
> 上次更新: 2024-01-10
> 更新原因: 新增搜索功能，调整 Header 布局
```

## 常见问题

### Q: 截图文件放在哪里?
A: 放在 `docs/screenshots/{PageName}/` 目录下，与 `docs/layout/` 平级。

### Q: 如何处理响应式布局?
A: 为每个断点分别截图，命名为 `overview-mobile.png`、`overview-tablet.png` 等。

### Q: 截图需要包含数据吗?
A: 建议使用模拟数据，保持截图的一致性和可重复性。

### Q: 如何标注动态内容?
A: 使用多张截图展示不同状态，或使用 GIF/视频格式。
