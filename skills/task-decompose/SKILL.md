name: task-decompose
description: 将复杂任务（>10文件或>3步骤）分解为并行子任务，使用agent执行
---

## 触发条件

任务预计涉及 >10 个文件或 >3 个步骤时触发。

## 执行步骤

1. 分析任务范围，确定可并行的子领域
2. 创建并行agent任务：

```
Agent 1: 探索和记录 [组件A] 的当前状态
- 列出所有相关文件
- 记录当前实现细节
- 识别风险和依赖

Agent 2: 探索和记录 [组件B] 的当前状态
- 同上

Agent 3: 识别所有需要变更的文件
- 按风险分类（高/中/低）
- 估计每个文件的变更范围
- 标记跨模块依赖
```

3. 等待所有agent报告完成
4. 汇总结果，向用户展示：
   - 总文件数
   - 按风险分类的文件列表
   - 建议的实施顺序
5. 等待用户确认后，分配实现阶段

## 禁止

- 不分解直接开始修改大量文件
- 不评估风险就并行修改有依赖的文件
