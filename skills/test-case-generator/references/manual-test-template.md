# 手工测试用例模板

## 基本格式

每个手工测试用例应包含以下要素：

```markdown
## TC-<编号>: <用例标题>

**优先级**：<高/中/低>
**前置条件**：
1. <前置条件1>
2. <前置条件2>

**测试步骤**：
| 步骤 | 操作 | 输入数据 | 预期结果 |
|------|------|----------|----------|
| 1 | <操作描述> | <输入值> | <预期输出> |
| 2 | <操作描述> | <输入值> | <预期输出> |

**预期结果**：
- <断言1>
- <断言2>

**备注**：<补充说明>
```

## 冒烟测试清单模板

```markdown
# 冒烟测试清单 - <模块名>

## 环境
- 版本：<版本号>
- 环境：<测试/预发/生产>
- 测试人：
- 测试日期：

## 测试项

| # | 功能 | 操作 | 预期结果 | 通过 |
|---|------|------|----------|------|
| 1 | 登录 | 输入正确账密登录 | 登录成功，跳转首页 | [ ] |
| 2 | 核心功能A | 执行操作X | 返回正确结果 | [ ] |
| 3 | 核心功能B | 执行操作Y | 返回正确结果 | [ ] |
```

## 黑盒测试用例模板

```markdown
## BT-<编号>: <接口/功能黑盒测试>

**测试对象**：<API路径或功能名称>
**测试方法**：<HTTP请求/界面操作/命令行>

### 正向测试

| # | 输入 | 预期输出 | 实际输出 | 通过 |
|---|------|----------|----------|------|
| 1 | <正常输入A> | <期望响应A> | | [ ] |
| 2 | <正常输入B> | <期望响应B> | | [ ] |

### 边界测试

| # | 输入 | 预期输出 | 实际输出 | 通过 |
|---|------|----------|----------|------|
| 1 | <空值> | <错误提示> | | [ ] |
| 2 | <最大值> | <正常处理> | | [ ] |
| 3 | <最小值> | <正常处理> | | [ ] |

### 异常测试

| # | 输入 | 预期输出 | 实际输出 | 通过 |
|---|------|----------|----------|------|
| 1 | <无效格式> | <错误码/提示> | | [ ] |
| 2 | <超范围值> | <错误码/提示> | | [ ] |
| 3 | <特殊字符> | <正确处理> | | [ ] |
```

## 回归测试清单模板

```markdown
# 回归测试 - <Bug标题>

**关联 Bug**：<Bug ID>
**修复版本**：<版本号>
**修复人**：<开发者>

### 复现验证

| # | 原始复现步骤 | 预期（已修复） | 通过 |
|---|-------------|---------------|------|
| 1 | <原Bug复现步骤> | <不应再出现> | [ ] |

### 影响范围验证

| # | 相关功能 | 测试操作 | 预期 | 通过 |
|---|---------|---------|------|------|
| 1 | <关联功能A> | <操作> | <预期> | [ ] |
| 2 | <关联功能B> | <操作> | <预期> | [ ] |
```

## 测试计划模板

```markdown
# 测试计划 - <项目/功能名>

## 1. 测试范围

### 包含
- <功能模块1>
- <功能模块2>

### 不包含
- <不在范围内的部分>

## 2. 测试策略

| 测试类型 | 覆盖范围 | 用例数 | 负责人 |
|----------|---------|--------|--------|
| 单元测试 | 核心逻辑 | - | 开发 |
| 集成测试 | 模块交互 | - | 开发 |
| 冒烟测试 | 核心路径 | - | QA |
| 功能测试 | 全部功能 | - | QA |
| 回归测试 | Bug修复 | - | QA |

## 3. 测试环境

| 环境 | 版本 | 用途 |
|------|------|------|
| 开发环境 | latest | 开发自测 |
| 测试环境 | v1.x | QA验证 |
| 预发环境 | v1.x | 上线前验证 |

## 4. 验收标准
- 所有冒烟测试通过
- 无 P0/P1 级 Bug
- P2 级 Bug 不超过 N 个
```
