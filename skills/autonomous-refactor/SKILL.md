## 触发条件

以下场景触发此skill：
- 大规模重构任务（>20文件或涉及多模块）
- 技术债务清理（h()→SFC迁移、错误处理现代化等）
- 用户明确说"使用自主重构流水线"或"/autonomous-refactor"

## Agent团队架构

### Agent 1: TEST_AGENT

**职责**：分析现有实现，生成全面测试套件

**执行步骤**：
1. 探索目标代码区域，记录所有公共API和行为
2. 生成单元测试覆盖核心逻辑
3. 生成Playwright/E2E测试覆盖用户交互路径
4. 标记已知bug为EXPECTED_FAIL（附注释说明）
5. 运行测试确认基线通过（已知失败除外）

**成功标准**：
- 测试覆盖率≥80%
- 所有非EXPECTED_FAIL测试通过
- 测试文档完整说明每个测试的目的

### Agent 2: REFACTOR_AGENT

**职责**：执行重构，每批次验证

**执行步骤**：
1. 按风险/依赖排序文件列表
2. 每5个文件为一个批次
3. 每个批次：重构→运行测试→全部通过才继续
4. 遇到意外失败：停止，spawn DEBUG_AGENT诊断
5. 记录每个批次的变更摘要

**成功标准**：
- 所有批次测试通过
- 无功能回归（对比基线测试）
- 代码质量提升（复杂度降低、可读性改善）

### Agent 3: VERIFY_AGENT

**职责**：跨仓库集成验证

**执行步骤**：
1. 运行完整集成测试套件
2. 验证Feishu通知流程
3. 验证Excel解析功能
4. 验证git hooks功能
5. 验证前端构建产物

**成功标准**：
- 所有集成测试通过
- 手动检查关键用户路径无异常
- 性能无显著退化

### Agent 4: DEBUG_AGENT（按需spawn）

**职责**：诊断和重试失败的批次

**触发条件**：
- REFACTOR_AGENT遇到意外测试失败
- VERIFY_AGENT发现集成问题

**执行步骤**：
1. 分析失败原因（测试问题vs实现问题）
2. 提出修正策略
3. 应用修正并重新运行测试
4. 如仍失败，升级报告给用户

## 执行流程

```
Phase 1: TEST_AGENT 生成测试套件
    ↓ 测试基线确认
Phase 2: REFACTOR_AGENT 分批重构
    ↓ 每批次通过
Phase 3: VERIFY_AGENT 集成验证
    ↓ 全部通过
完成：汇总报告（测试数/重构文件数/验证结果）
```

## 用户交互规则

- **执行期间不提问**：agent自主决策，不中断用户
- **仅在以下情况升级**：
  - DEBUG_AGENT无法解决失败
  - 需要用户确认架构决策
  - 发现未预期的大规模影响
- **最终报告包含**：
  - 编写的测试数量
  - 重构的提交记录
  - 验证结果
  - 需要人工决策的事项

## 禁止

- 不生成测试直接重构
- 遇到失败不停止继续下一批次
- 未验证就声称成功
