## MemPalace 记忆系统

MemPalace 是 AI 记忆存储系统，以 Claude Code 插件形式运行。

### 两套独立环境

| 环境 | Python | mempalace | chromadb | palace 数据 |
|------|--------|-----------|----------|-------------|
| Windows MCP | 3.12 | 3.3.5 | 1.5.9 | `C:/Users/v-wangtengda/.mempalace/palace/` |
| WSL Ubuntu-24.04 | 3.12 | 3.3.4 | 1.5.8 | `/root/.mempalace/palace/` |

两套环境完全独立：不同的 Python、不同的 palace 数据、不同的源码。

### Windows 侧配置（2026-05-11 维护）

- **Python 版本**：MCP 实际使用 Python 3.12（不是 3.14），路径 `C:\Users\v-wangtengda\AppData\Local\Programs\Python\Python312\`
- **Embedding 模型**：已从默认 `all-MiniLM-L6-v2`（英文，384维）切换为 `BAAI/bge-base-zh-v1.5`（中文优化，768维）
- **源码修改**：
  - `Python312\Lib\site-packages\mempalace\embedding.py` — 修改 `get_embedding_function()` 使用 `BAAI/bge-base-zh-v1.5`
  - `Python312\Lib\site-packages\mempalace\hooks_cli.py` — 修复 `_wing_from_transcript_path()` 正则表达式支持 Windows 盘符路径（`D--`、`C--`）
- **关键依赖**：`chroma-hnswlib==0.7.6`（不能装原版 `hnswlib`，否则 `file_handle_count` 缺失导致 ChromaDB 崩溃）
- **旧数据已清空**：切换 embedding 后向量维度不兼容，旧 palace 数据已备份到 `palace_backup_old_embedding/`
- **重建索引**：需执行 `mempalace mine <项目目录>` 重新导入数据
- **Hooks 自动保存**：Stop/PreCompact hooks 已修复路径解析，现在能正确保存 Claude Code 对话到 `wing_func`

### Windows 侧已知问题

- **MCP 中文搜索编码 bug**：mempalace 3.3.5 MCP 层中文搜索有 `TextEncodeInput must be Union[TextInputSequence, ...]` 错误，CLI 搜索正常
- **UTF-8 编码**：Windows 终端默认 GBK，需设置 `$env:PYTHONUTF8=1` 避免 mempalace 输出乱码
- **MCP 插件重启**：修改依赖后需重启 Claude Code 会话，插件进程不会自动重载
- **Python 3.14 不相关**：MCP 使用 Python 3.12，py 命令调用的是 Python 3.14，两者独立

### WSL 侧

- `mempalace.yaml` 配置了 `wing: config`，有 skip/include patterns
- 通过 `wsl -d Ubuntu-24.04 -- python3 -m mempalace` 使用
